02 // Context

Enterprise Tax Compliance Is Not Optional Software

The Taxation Platform was a complex enterprise system for large corporate clients — managing tax calculations, compliance tracking, and multi-role financial reporting. Its Windows-only architecture had become a barrier: technically constrained, visually outdated, and burdened with usability issues that cost clients time and trust.

The mandate was clear: rebuild it as a modern cloud web platform. But the transformation went far deeper than screens and code — it required rebuilding the team's processes, alignment, and shared understanding from scratch.

Core insight

For enterprise clients, a taxation platform isn't optional software — it's the system that ensures regulatory compliance for entire organizations. The redesign had to treat it with that gravity.

03 // The Problem

Legacy Software Doesn't Fail in Isolation

When I joined the project, I found two problems compounding each other. The product problem was visible: a Windows-only system with no role-specific navigation, no design system, where every module felt different and enterprise users bore the training cost — but the organizational problem was the deeper threat.

The distributed team had no shared vision, no role clarity, no defined contribution model, and requirements that shifted constantly from the client side. Individual quality work was being blocked by team misalignment.

05 // Discovery

Mapping the Full Landscape Before Designing Anything

Before touching a single screen, I mapped the full landscape — reviewing product documentation, video walkthroughs, and live demo sessions to build a working model of the existing system. Then I conducted stakeholder interviews to validate business goals, surface unstated expectations, and establish a shared definition of success.

From this foundation, I built: a product mindmap showing the as-is to to-be transformation, role-based user maps for each distinct user type, customer journey maps per role, and both global and role-specific sitemaps.

06 // Key Insights

Five Realities That Shaped Every Decision

  • Role complexity was the root cause. The platform served fundamentally different user roles — one navigation for all forced everyone to navigate complexity intended for someone else.
  • The delivery problem was organizational, not technical. Misaligned expectations generated friction that good screens alone couldn't resolve. Teams needed alignment before they needed designs.
  • Training cost was a normalized pain. The legacy product had normalized a high training burden. Reducing it required rebuilding information architecture from the bottom up, not just reskinning.
  • A design system was a prerequisite, not a deliverable. Shipping screens without a shared system creates visual divergence that compounds with every sprint.
  • Rigid roadmaps were a liability. Constantly evolving client requirements made rigid planning dangerous. Modular roadmaps — global plus per-module — allowed adaptation without chaos.
07 // Strategy

Build Foundations Before Features

The strategic direction was clear: align the team, map the architecture, establish the design system — then ship screens. Six phases from discovery to accessibility integration, each building on the last.

Team Alignment Workshop

I noticed early that individual quality work was being blocked by team misalignment. Rather than pushing forward in isolation, I paused design delivery to facilitate a structured alignment workshop — friction dropped sharply, and every team member gained a clear backlog with an understood contribution model.

08 // Key Decisions

Three Bets That Shaped the Outcome

01

Outcome

02
Design System Before Screens
Challenge
Pressure to ship UI quickly. No shared components, tokens, or patterns existed — each module would diverge visually.
Outcome

03
Role-Specific Navigation
Challenge
Enterprise convention was one navigation for all users. But the platform served fundamentally different roles with different workflows.
Outcome
Each role got navigation scaled to their actual use — reducing cognitive load, training overhead, and the 60% training time improvement.
09 // Solution

A Complete Cloud Platform with a Living Design System

The final delivery was a full cloud-based web platform replacing the legacy Windows desktop. New brand identity, role-specific interfaces, modern responsive layout, and a comprehensive design system that enabled parallel development across three major modules.

The Design System

A unified component library with grid, spacing tokens, navigation models, scrolling behavior, interaction patterns, and fully documented states.

Documentation & Knowledge Transfer

A documentation ecosystem designed for scale: DS documentation with component anatomy, states, and usage guidelines; an onboarding package for new design contributors; and accessibility guidelines built into the system itself.
